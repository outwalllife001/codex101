MCP 集成指南

Model Context Protocol (MCP) 将模型连接到工具和上下文。使用它让 Codex 访问第三方文档或开发者工具。

什么是 MCP？

MCP 是一种开放标准协议，允许 AI 模型通过标准化的方式访问外部工具和数据源。它类似于 AI 应用的"USB-C"接口。

MCP 服务器类型

STDIO 服务器

作为本地进程运行的服务器（通过命令启动）：

  • 使用环境变量配置
  • 适合本地工具集成

Streamable HTTP 服务器

通过网络地址访问的服务器：

  • Bearer token 认证
  • OAuth 认证
  • 适合云服务集成

配置 MCP

Codex 将 MCP 配置存储在 config.toml 中。

方法 1：使用 CLI

# 添加 MCP 服务器
codex mcp add  --env VAR1=VALUE1 --env VAR2=VALUE2 -- 

# 示例：添加 Context7
codex mcp add context7 -- npx -y @upstash/context7-mcp

# 添加 Playwright
codex mcp add playwright npx @playwright/mcp@latest

# 查看所有命令
codex mcp --help

方法 2：手动编辑 config.toml

STDIO 服务器示例

[mcp_servers.context7]
command = "npx"
args = ["-y", "@upstash/context7-mcp"]

[mcp_servers.context7.env]
MY_ENV_VAR = "MY_ENV_VALUE"

HTTP 服务器示例

[mcp_servers.figma]
url = "https://mcp.figma.com/mcp"
bearer_token_env_var = "FIGMA_OAUTH_TOKEN"
http_headers = { "X-Figma-Region" = "us-east-1" }

[mcp_servers.chrome_devtools]
url = "http://localhost:3000/mcp"
enabled_tools = ["open", "screenshot"]
disabled_tools = ["close"]
startup_timeout_sec = 20
tool_timeout_sec = 45

MCP Integration Guide

Model Context Protocol (MCP) connects models to tools and context. Use it to let Codex access third-party documentation or developer tools.

What is MCP?

MCP is an open standard protocol that allows AI models to access external tools and data sources in a standardized way. It is like a "USB-C" interface for AI applications.

MCP Server Types

STDIO Servers

Servers that run as local processes (started via commands):

  • Configured using environment variables
  • Suitable for local tool integration

Streamable HTTP Servers

Servers accessed via network addresses:

  • Bearer token authentication
  • OAuth authentication
  • Suitable for cloud service integration

MCP 統合ガイド

Model Context Protocol（MCP）は、モデルとツールやコンテキストを接続するための仕組みです。これを使うと、Codex がサードパーティ製ドキュメントや開発者ツールにアクセスできるようになります。

MCP とは？

MCP は、AI モデルが外部ツールやデータソースに標準化された方法でアクセスできるようにするオープン標準プロトコルです。AI アプリケーション向けの「USB-C インターフェース」のようなものと考えると分かりやすいです。

MCP サーバーの種類

STDIO サーバー

コマンドで起動するローカルプロセスとして動くサーバーです：

  • 環境変数を使って設定する
  • ローカルツール統合に向いている

Streamable HTTP サーバー

ネットワークアドレス経由で利用するサーバーです：

  • Bearer トークン認証
  • OAuth 認証
  • クラウドサービス統合に向いている

Guia de Integracion MCP

Model Context Protocol (MCP) conecta modelos con herramientas y contexto. Sirve para que Codex pueda acceder a documentacion de terceros o herramientas de desarrollo.

Que es MCP?

MCP es un protocolo abierto y estandar que permite a los modelos de IA acceder a herramientas externas y fuentes de datos de forma uniforme. Es como una interfaz "USB-C" para aplicaciones de IA.

Tipos de servidores MCP

Servidores STDIO

Servidores que se ejecutan como procesos locales (arrancados mediante comandos):

  • Se configuran con variables de entorno
  • Son adecuados para integrar herramientas locales

Servidores HTTP streamable

Servidores accesibles mediante direcciones de red:

  • Autenticacion con bearer token
  • Autenticacion OAuth
  • Adecuados para integrar servicios en la nube
