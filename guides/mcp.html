MCP 集成指南

Model Context Protocol (MCP) 将模型连接到工具和上下文。使用它让 Codex 访问第三方文档或开发者工具。

什么是 MCP？

MCP 是一种开放标准协议，允许 AI 模型通过标准化的方式访问外部工具和数据源。它类似于 AI 应用的"USB-C"接口。

MCP 服务器类型

STDIO 服务器

作为本地进程运行的服务器（通过命令启动）：

  • 使用环境变量配置
  • 适合本地工具集成

Streamable HTTP 服务器

通过网络地址访问的服务器：

  • Bearer token 认证
  • OAuth 认证
  • 适合云服务集成

配置 MCP

Codex 将 MCP 配置存储在 config.toml 中。

方法 1：使用 CLI

# 添加 MCP 服务器
codex mcp add  --env VAR1=VALUE1 --env VAR2=VALUE2 -- 

# 示例：添加 Context7
codex mcp add context7 -- npx -y @upstash/context7-mcp

# 添加 Playwright
codex mcp add playwright npx @playwright/mcp@latest

# 查看所有命令
codex mcp --help

方法 2：手动编辑 config.toml

STDIO 服务器示例

[mcp_servers.context7]
command = "npx"
args = ["-y", "@upstash/context7-mcp"]

[mcp_servers.context7.env]
MY_ENV_VAR = "MY_ENV_VALUE"

HTTP 服务器示例

[mcp_servers.figma]
url = "https://mcp.figma.com/mcp"
bearer_token_env_var = "FIGMA_OAUTH_TOKEN"
http_headers = { "X-Figma-Region" = "us-east-1" }

[mcp_servers.chrome_devtools]
url = "http://localhost:3000/mcp"
enabled_tools = ["open", "screenshot"]
disabled_tools = ["close"]
startup_timeout_sec = 20
tool_timeout_sec = 45

常用 MCP 服务器

服务器 用途 安装命令
Context7 开发者文档搜索 codex mcp add context7 -- npx -y @upstash/context7-mcp
Playwright 浏览器自动化 codex mcp add playwright npx @playwright/mcp@latest
Chrome DevTools Chrome 调试 HTTP 服务器
GitHub PR/Issue 管理 codex mcp add github --env GITHUB_TOKEN=xxx -- npx -y @modelcontextprotocol/server-github
Sentry 错误日志 HTTP 服务器

高级配置

工具白名单/黑名单

[mcp_servers.myserver]
enabled_tools = ["tool1", "tool2"]
disabled_tools = ["dangerous_tool"]

超时设置

startup_timeout_sec = 10   # 启动超时
tool_timeout_sec = 60       # 工具执行超时

OAuth 配置

# 配置回调端口
mcp_oauth_callback_port = 5555

# 或自定义回调 URL
mcp_oauth_callback_url = "https://devbox.example.internal/callback"

# 登录
codex mcp login

故障排查

检查连接状态

/mcp

预期输出：

Connected MCP Servers:
  ✓ context7 - Ready
  ✓ playwright - Ready
  ✗ github - Error: Connection refused

常见问题

  • 服务器未启动：检查命令是否正确
  • 认证失败：验证 token 或重新登录
  • 工具未显示：重启 Codex 或检查配置
