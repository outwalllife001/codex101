AGENTS.md 完全指南

AGENTS.md 是 Codex 的"README for Agents"，它让 Codex 了解如何在特定仓库中工作。 这是提高 Codex 效率和一致性的最重要配置。

为什么需要 AGENTS.md？

当提示模式有效后，下一步是停止手动重复它。这就是 AGENTS.md 的作用。

一个好的 AGENTS.md 帮助 Codex：

  • 理解仓库结构和重要目录
  • 知道如何运行、构建和测试项目
  • 遵循团队的工程规范和 PR 期望
  • 了解约束和禁止事项
  • 明白"完成"意味着什么以及如何验证工作

层级结构

Codex 会在以下位置查找 AGENTS.md（按优先级排序）：

  1. 当前目录: 最接近工作文件的 AGENTS.md
  2. 项目根目录: 仓库级别的共享标准
  3. ~/.codex/: 个人默认配置
优先级规则: 更具体的文件（离当前目录更近）会覆盖更通用的配置。

快速开始

使用 /init 命令快速生成 AGENTS.md 脚手架：

codex> /init

## 代码规范

### 命名约定
- 组件使用 PascalCase: MyComponent.tsx
- 工具函数使用 camelCase: formatDate.ts
- 常量使用 UPPER_SNAKE_CASE

### 文件组织
- 每个组件一个文件夹，包含 index.tsx 和 styles.css
- 共享类型放在 src/types/
- 工具函数放在 src/utils/

### 导入顺序
1. 外部依赖
2. 内部模块
3. 相对路径导入
4. 类型导入

## 约束与禁止事项

- ❌ 不要使用 any 类型
- ❌ 不要提交 console.log
- ❌ 不要在组件中直接调用 API
- ✅ 所有函数必须有返回类型注解
- ✅ 所有组件必须有 PropTypes 或接口定义

## PR 期望

- 提交前运行完整的测试套件
- 确保代码通过 lint 检查
- 添加相关的单元测试
- 更新相关文档

## 完成定义

任务完成的标准：
1. 代码实现完成
2. 测试通过（新增功能要有测试覆盖）
3. 类型检查通过
4. Lint 检查通过
5. 相关文档已更新

## 调试指南

### 常见问题
- 启动失败：检查 .env 文件是否配置正确
- 测试失败：确认数据库连接是否正常
- 构建失败：检查是否有未解决的类型错误

### 日志位置
- 应用日志: logs/app.log
- 错误日志: logs/error.log

最佳实践

1. 保持简洁

简短准确的 AGENTS.md 比充满模糊规则的长文件更有用。从基础开始，只在注意到重复错误时才添加新规则。

2. 基于实际摩擦

当 Codex 犯同样的错误两次时，询问它进行回顾并更新 AGENTS.md。指导应始终基于实际的摩擦。

3. 引用其他文件

如果 AGENTS.md 变得太大，保持主文件简洁，并引用任务特定的 markdown 文件：

## 代码审查
参见: ./docs/code_review.md

## 架构决策
参见: ./docs/architecture.md

4. 与团队共享

将 AGENTS.md 提交到版本控制，确保所有开发者和 Agent 共享相同的流程。

针对技术栈的示例

React + TypeScript

## 技术栈
- React 18 with hooks
- TypeScript (严格模式)
- Vite for build
- Tailwind CSS for styling
- React Query for data fetching
- Zustand for state management

## 组件规范
- 使用函数组件和 hooks
- Props 接口必须以 'Props' 结尾
- 使用 named exports
- 样式使用 Tailwind classes

## 数据获取
- 所有 API 调用必须通过 React Query
- 使用自定义 hooks 封装数据逻辑
- 错误处理使用 ErrorBoundary

Python + FastAPI

## 技术栈
- Python 3.11+
- FastAPI
- Pydantic for validation
- SQLAlchemy for ORM
- pytest for testing

## 项目结构
- 路由放在 routers/ 目录
- 模型放在 models/ 目录
-  schemas 放在 schemas/ 目录
- 测试放在 tests/ 目录，镜像 src 结构

## 代码规范
- 使用 type hints  everywhere
- 使用 Pydantic models 进行请求/响应验证
- 数据库操作使用依赖注入
- 异步函数使用 async/await

故障排查

如果 Codex 似乎没有遵循 AGENTS.md：

  1. 确认文件位置正确（使用 /debug-config 查看加载的配置）
  2. 检查文件格式（应为有效的 Markdown）
  3. 确保规则清晰明确
  4. 尝试将关键规则放在文件前面
