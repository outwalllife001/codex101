AGENTS.md 完全指南

AGENTS.md 是 Codex 的"README for Agents"，它让 Codex 了解如何在特定仓库中工作。 这是提高 Codex 效率和一致性的最重要配置。

为什么需要 AGENTS.md？

当提示模式有效后，下一步是停止手动重复它。这就是 AGENTS.md 的作用。

一个好的 AGENTS.md 帮助 Codex：

  • 理解仓库结构和重要目录
  • 知道如何运行、构建和测试项目
  • 遵循团队的工程规范和 PR 期望
  • 了解约束和禁止事项
  • 明白"完成"意味着什么以及如何验证工作

层级结构

Codex 会在以下位置查找 AGENTS.md（按优先级排序）：

  1. 当前目录: 最接近工作文件的 AGENTS.md
  2. 项目根目录: 仓库级别的共享标准
  3. ~/.codex/: 个人默认配置
优先级规则: 更具体的文件（离当前目录更近）会覆盖更通用的配置。

## 代码规范

### 命名约定
- 组件使用 PascalCase: MyComponent.tsx
- 工具函数使用 camelCase: formatDate.ts
- 常量使用 UPPER_SNAKE_CASE

### 文件组织
- 每个组件一个文件夹，包含 index.tsx 和 styles.css
- 共享类型放在 src/types/
- 工具函数放在 src/utils/

### 导入顺序
1. 外部依赖
2. 内部模块
3. 相对路径导入
4. 类型导入

## 约束与禁止事项

- ❌ 不要使用 any 类型
- ❌ 不要提交 console.log
- ❌ 不要在组件中直接调用 API
- ✅ 所有函数必须有返回类型注解
- ✅ 所有组件必须有 PropTypes 或接口定义

## PR 期望

- 提交前运行完整的测试套件
- 确保代码通过 lint 检查
- 添加相关的单元测试
- 更新相关文档

## 完成定义

任务完成的标准：
1. 代码实现完成
2. 测试通过（新增功能要有测试覆盖）
3. 类型检查通过
4. Lint 检查通过
5. 相关文档已更新

## 调试指南

### 常见问题
- 启动失败：检查 .env 文件是否配置正确
- 测试失败：确认数据库连接是否正常
- 构建失败：检查是否有未解决的类型错误

### 日志位置
- 应用日志: logs/app.log
- 错误日志: logs/error.log

最佳实践

1. 保持简洁

简短准确的 AGENTS.md 比充满模糊规则的长文件更有用。从基础开始，只在注意到重复错误时才添加新规则。

2. 基于实际摩擦

当 Codex 犯同样的错误两次时，询问它进行回顾并更新 AGENTS.md。指导应始终基于实际的摩擦。

3. 引用其他文件

如果 AGENTS.md 变得太大，保持主文件简洁，并引用任务特定的 markdown 文件：

## 代码审查
参见: ./docs/code_review.md

## 架构决策
参见: ./docs/architecture.md

4. 与团队共享

将 AGENTS.md 提交到版本控制，确保所有开发者和 Agent 共享相同的流程。

AGENTS.md Complete Guide

AGENTS.md is Codex's "README for Agents" -- it tells Codex how to work in a specific repository. This is the most important configuration for improving Codex's efficiency and consistency.

Why Do You Need AGENTS.md?

Once you find a prompting pattern that works, the next step is to stop repeating it manually. That's what AGENTS.md is for.

A good AGENTS.md helps Codex:

  • Understand repository structure and important directories
  • Know how to run, build, and test the project
  • Follow the team's engineering standards and PR expectations
  • Be aware of constraints and prohibited actions
  • Understand what "done" means and how to verify work

Hierarchy

Codex looks for AGENTS.md in the following locations (by priority):

  1. Current directory: The AGENTS.md closest to the working files
  2. Project root: Repository-level shared standards
  3. ~/.codex/: Personal default configuration
Priority rule: More specific files (closer to the current directory) override more general configurations.

Best Practices

1. Keep It Concise

A short, precise AGENTS.md is more useful than a long file full of vague rules. Start with the basics and only add new rules when you notice repeated mistakes.

2. Base It on Real Friction

When Codex makes the same mistake twice, ask it to do a retrospective and update AGENTS.md. Guidance should always be based on actual friction.

3. Reference Other Files

If AGENTS.md gets too large, keep the main file concise and reference task-specific markdown files:

## Code Review
See: ./docs/code_review.md

## Architecture Decisions
See: ./docs/architecture.md

4. Share with the Team

Commit AGENTS.md to version control to ensure all developers and Agents share the same workflow.

AGENTS.md 完全ガイド

AGENTS.md は Codex にとっての「Agent 向け README」です。特定のリポジトリでどう振る舞うべきかを Codex に伝えます。 Codex の効率と一貫性を高めるうえで、最も重要な設定ファイルです。

なぜ AGENTS.md が必要なのか？

うまく機能するプロンプトの型が見つかったら、次の段階はそれを毎回手で繰り返さないことです。その役割を担うのが AGENTS.md です。

良い AGENTS.md は Codex に次のことを理解させます：

  • リポジトリ構造と重要なディレクトリ
  • プロジェクトの実行・ビルド・テスト方法
  • チームのエンジニアリング基準と PR の期待値
  • 制約事項や禁止事項
  • 「完了」とは何か、どう検証するか

階層

Codex は次の場所で AGENTS.md を探します（優先順）：

  1. 現在のディレクトリ: 作業ファイルに最も近い AGENTS.md
  2. プロジェクトルート: リポジトリ全体で共有する基準
  3. ~/.codex/: 個人のデフォルト設定
優先ルール: より具体的なファイル（現在の作業ディレクトリに近いもの）が、より一般的な設定を上書きします。

ベストプラクティス

1. 簡潔に保つ

曖昧なルールが大量に並んだ長文より、短く正確な AGENTS.md のほうが役立ちます。まずは基本から書き始め、同じミスが繰り返されたときにだけルールを足してください。

2. 実際の摩擦を起点にする

Codex が同じ失敗を 2 回したら、レトロスペクティブを行わせて AGENTS.md を更新します。ガイダンスは実際に起きた摩擦をもとに書くべきです。

3. 他ファイルを参照する

AGENTS.md が大きくなりすぎたら、メインファイルは簡潔に保ち、タスク別の Markdown を参照するようにします：

## Code Review
See: ./docs/code_review.md

## Architecture Decisions
See: ./docs/architecture.md

4. チームで共有する

AGENTS.md をバージョン管理に含めれば、開発者と Agent 全員が同じワークフローを共有できます。

Guia Completa de AGENTS.md

AGENTS.md es el "README para Agents" de Codex: le indica como debe trabajar dentro de un repositorio concreto. Es la configuracion mas importante para mejorar la eficiencia y la consistencia de Codex.

Por que necesitas AGENTS.md?

Cuando encuentras un patron de prompting que funciona, el siguiente paso es dejar de repetirlo manualmente. Para eso sirve AGENTS.md.

Un buen AGENTS.md ayuda a Codex a:

  • Entender la estructura del repositorio y los directorios importantes
  • Saber como ejecutar, compilar y probar el proyecto
  • Seguir los estandares de ingenieria del equipo y las expectativas de PR
  • Tener presentes las restricciones y acciones prohibidas
  • Entender que significa "terminado" y como verificarlo

Jerarquia

Codex busca AGENTS.md en las siguientes ubicaciones (por prioridad):

  1. Directorio actual: El AGENTS.md mas cercano a los archivos en los que trabajas
  2. Raiz del proyecto: Estandares compartidos a nivel de repositorio
  3. ~/.codex/: Configuracion personal por defecto
Regla de prioridad: Los archivos mas especificos (mas cercanos al directorio actual) sobrescriben las configuraciones mas generales.

Buenas practicas

1. Mantenlo conciso

Un AGENTS.md corto y preciso es mas util que un archivo largo lleno de reglas vagas. Empieza por lo basico y solo agrega nuevas reglas cuando veas errores repetidos.

2. Basalo en fricciones reales

Cuando Codex cometa el mismo error dos veces, pidelo una retrospectiva y actualiza AGENTS.md. Las indicaciones deben nacer de problemas reales.

3. Referencia otros archivos

Si AGENTS.md crece demasiado, manten el archivo principal conciso y referencia otros markdown especificos por tarea:

## Code Review
See: ./docs/code_review.md

## Architecture Decisions

4. Compartelo con el equipo

Versionar AGENTS.md permite que todos los desarrolladores y Agents trabajen con el mismo flujo.
