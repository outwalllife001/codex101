Skills 扩展

Skills 让 Codex 能够执行特定任务的工作流。一个 Skill 打包了指令、资源和可选脚本，使 Codex 能够可靠地完成特定任务。

什么是 Skill？

Skill 是 Codex 的扩展机制，它允许你：

  • 封装重复任务的指令
  • 定义标准化的工作流
  • 与团队共享最佳实践
  • 连接外部工具和服务

Skill 结构

my-skill/
├── SKILL.md          # 必需：指令和元数据
├── scripts/          # 可选：可执行代码
├── references/       # 可选：文档
├── assets/           # 可选：模板、资源
└── agents/
    └── openai.yaml   # 可选：UI 配置

SKILL.md 格式

---
name: security-audit
description: 扫描代码库查找安全漏洞，包括 SQL 注入、XSS、硬编码密钥
---

# Security Audit

请扫描当前代码库，查找以下漏洞：

1. SQL 注入风险
2. 硬编码的密钥或 token
3. XSS 漏洞
4. 不安全的依赖

生成 Markdown 格式的报告，包含：
- 问题严重程度（Critical/High/Medium/Low）
- 文件位置和代码片段
- 修复建议

使用 Skills

显式调用

$security-audit

或

/skills
# 然后从列表中选择

隐式调用

Codex 可以根据任务描述自动选择 Skill：

> 扫描一下代码的安全问题

安装 Skills

使用内置安装器

$skill-installer linear

Skill 发现位置

Codex 从以下位置扫描 Skills：

  1. 仓库的 .agents/skills/ 目录
  2. 用户目录 ~/.codex/skills/
  3. 系统目录（管理员配置）

创建自定义 Skill

步骤 1：使用创建器

$skill-creator

创建器会询问：

  • Skill 的作用
  • 何时应该触发
  • 纯指令还是包含脚本

步骤 2：手动创建

mkdir -p ~/.codex/skills/my-skill
cd ~/.codex/skills/my-skill
cat > SKILL.md << 'EOF'
---
name: my-skill
description: 描述 Skill 应该何时触发
---

# My Skill

指令内容...
EOF

常用内置 Skills

Skill 用途
$skill-creator 创建新 Skill
$skill-installer 安装社区 Skills
$linear 项目管理集成

配置 UI 元数据

在 agents/openai.yaml 中配置：

interface:
  display_name: "安全审计"
  short_description: "扫描代码安全漏洞"
  icon_small: "./assets/shield.svg"
  brand_color: "#ef4444"
  default_prompt: "请扫描代码库的安全问题"

policy:
  allow_implicit_invocation: true

dependencies:
  tools:
    - type: "mcp"
      value: "securityScanner"

禁用 Skill

在 ~/.codex/config.toml 中：

[[skills.config]]
path = "/path/to/skill/SKILL.md"
enabled = false

最佳实践

  • 每个 Skill 专注于一个任务
  • 优先使用指令而非脚本
  • 编写明确的输入输出说明
  • 测试触发描述确保匹配正确
