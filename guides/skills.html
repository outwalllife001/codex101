Skills 扩展

Skills 让 Codex 能够执行特定任务的工作流。一个 Skill 打包了指令、资源和可选脚本，使 Codex 能够可靠地完成特定任务。

什么是 Skill？

Skill 是 Codex 的扩展机制，它允许你：

  • 封装重复任务的指令
  • 定义标准化的工作流
  • 与团队共享最佳实践
  • 连接外部工具和服务

Skill 结构

my-skill/
├── SKILL.md          # 必需：指令和元数据
├── scripts/          # 可选：可执行代码
├── references/       # 可选：文档
├── assets/           # 可选：模板、资源
└── agents/
    └── openai.yaml   # 可选：UI 配置

SKILL.md 格式

---
name: security-audit
description: 扫描代码库查找安全漏洞，包括 SQL 注入、XSS、硬编码密钥
---

# Security Audit

请扫描当前代码库，查找以下漏洞：

1. SQL 注入风险
2. 硬编码的密钥或 token
3. XSS 漏洞
4. 不安全的依赖

生成 Markdown 格式的报告，包含：
- 问题严重程度（Critical/High/Medium/Low）
- 文件位置和代码片段
- 修复建议

使用 Skills

显式调用

$security-audit

或

/skills
# 然后从列表中选择

隐式调用

Codex 可以根据任务描述自动选择 Skill：

> 扫描一下代码的安全问题

安装 Skills

使用内置安装器

$skill-installer linear

Skill 发现位置

Codex 从以下位置扫描 Skills：

  1. 仓库的 .agents/skills/ 目录
  2. 用户目录 ~/.codex/skills/
  3. 系统目录（管理员配置）

创建自定义 Skill

步骤 1：使用创建器

$skill-creator

创建器会询问：

  • Skill 的作用
  • 何时应该触发
  • 纯指令还是包含脚本

步骤 2：手动创建

mkdir -p ~/.codex/skills/my-skill
cd ~/.codex/skills/my-skill
cat > SKILL.md << 'EOF'
---
name: my-skill
description: 描述 Skill 应该何时触发
---

# My Skill

指令内容...
EOF

常用内置 Skills

Skill 用途
$skill-creator 创建新 Skill
$skill-installer 安装社区 Skills
$linear 项目管理集成

配置 UI 元数据

在 agents/openai.yaml 中配置：

interface:
  display_name: "安全审计"
  short_description: "扫描代码安全漏洞"
  icon_small: "./assets/shield.svg"
  brand_color: "#ef4444"
  default_prompt: "请扫描代码库的安全问题"

policy:
  allow_implicit_invocation: true

dependencies:
  tools:
    - type: "mcp"
      value: "securityScanner"

禁用 Skill

在 ~/.codex/config.toml 中：

[[skills.config]]
path = "/path/to/skill/SKILL.md"
enabled = false

最佳实践

  • 每个 Skill 专注于一个任务
  • 优先使用指令而非脚本
  • 编写明确的输入输出说明
  • 测试触发描述确保匹配正确

Skills Extension

Skills enable Codex to execute task-specific workflows. A Skill packages instructions, resources, and optional scripts, allowing Codex to reliably complete specific tasks.

What Is a Skill?

A Skill is Codex's extension mechanism that allows you to:

  • Encapsulate instructions for repetitive tasks
  • Define standardized workflows
  • Share best practices with your team
  • Connect external tools and services

Skill Structure

my-skill/
├── SKILL.md          # Required: instructions and metadata
├── scripts/          # Optional: executable code
├── references/       # Optional: documentation
├── assets/           # Optional: templates, resources
└── agents/
    └── openai.yaml   # Optional: UI configuration

SKILL.md Format

---
name: security-audit
description: Scan the codebase for security vulnerabilities including SQL injection, XSS, hardcoded secrets
---

# Security Audit

Please scan the current codebase for the following vulnerabilities:

1. SQL injection risks
2. Hardcoded secrets or tokens
3. XSS vulnerabilities
4. Insecure dependencies

Generate a Markdown report containing:
- Issue severity (Critical/High/Medium/Low)
- File location and code snippets
- Fix recommendations

Using Skills

Explicit Invocation

$security-audit

Or

/skills
# Then select from the list

Implicit Invocation

Codex can automatically select a Skill based on the task description:

> Scan the code for security issues

Installing Skills

Using the Built-in Installer

$skill-installer linear

Skill Discovery Locations

Codex scans for Skills in the following locations:

  1. The repository's .agents/skills/ directory
  2. User directory ~/.codex/skills/
  3. System directory (admin configuration)

Best Practices

  • Each Skill should focus on a single task
  • Prefer instructions over scripts
  • Write clear input/output specifications
  • Test trigger descriptions to ensure correct matching

Skills 拡張

Skills を使うと、Codex にタスク特化のワークフローを実行させられます。Skill は手順・参照資料・必要に応じてスクリプトをまとめてパッケージ化し、特定の仕事を安定してこなせるようにします。

Skill とは？

Skill は Codex の拡張機構で、次のことを可能にします：

  • 繰り返し作業の手順をカプセル化する
  • 標準化されたワークフローを定義する
  • チーム内でベストプラクティスを共有する
  • 外部ツールやサービスと連携する

  • トリガー説明をテストして正しくマッチするか確認する

Extension Skills

Las Skills permiten que Codex ejecute flujos de trabajo especificos para una tarea. Una Skill empaqueta instrucciones, recursos y, si hace falta, scripts opcionales para que Codex complete trabajos concretos con mas fiabilidad.

Que es una Skill?

Una Skill es el mecanismo de extension de Codex y te permite:

  • Encapsular instrucciones para tareas repetitivas
  • Definir flujos de trabajo estandarizados
  • Compartir buenas practicas con tu equipo
  • Conectar herramientas y servicios externos

Estructura de una Skill

my-skill/
├── SKILL.md          # Requerido: instrucciones y metadatos
├── scripts/          # Opcional: codigo ejecutable
├── references/       # Opcional: documentacion
├── assets/           # Opcional: plantillas y recursos
└── agents/
    └── openai.yaml   # Opcional: configuracion de UI

Formato de SKILL.md

---
name: security-audit
description: Scan the codebase for security vulnerabilities including SQL injection, XSS, hardcoded secrets
---

# Security Audit

Please scan the current codebase for the following vulnerabilities:

1. SQL injection risks
2. Hardcoded secrets or tokens
3. XSS vulnerabilities
4. Insecure dependencies

Generate a Markdown report containing:
- Issue severity (Critical/High/Medium/Low)
- File location and code snippets
- Fix recommendations

Uso de Skills

Invocacion explicita

$security-audit

O

/skills
# Then select from the list

Invocacion implicita

Codex puede seleccionar automaticamente una Skill a partir de la descripcion de la tarea:

> Revisa el codigo en busca de problemas de seguridad

Instalar Skills

Usando el instalador integrado

$skill-installer linear

Ubicaciones de descubrimiento

Codex busca Skills en estas ubicaciones:

  1. El directorio .agents/skills/ del repositorio
  2. El directorio de usuario ~/.codex/skills/
  3. El directorio del sistema (configuracion de administrador)

Buenas practicas

  • Cada Skill debe centrarse en una sola tarea
  • Prioriza instrucciones antes que scripts
  • Escribe especificaciones claras de entrada y salida
  • Prueba las descripciones de activacion para asegurar una coincidencia correcta
