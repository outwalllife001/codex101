核心概念

理解 Codex 的核心概念是高效使用它的关键。本页面介绍 Agent 模式、审批系统、工作流等基础概念。

什么是 Codex？

OpenAI Codex 是一个 AI 编程助手，它基于 GPT-5 系列模型，专门用于软件工程任务。 Codex 不仅仅是一个代码生成器，它是一个Agent（智能体），能够：

  • 阅读和理解你的代码库
  • 创建和编辑文件
  • 运行命令和测试
  • 进行代码审查
  • 在沙箱环境中安全执行代码

工作模式

Agent 模式（默认）

在 Agent 模式下，Codex 可以读取文件、运行命令、编写代码。它会在执行需要批准的操作前询问你。 这是最常用的模式，适合大多数开发任务。

Read Only 模式

Read Only 模式保持 Codex 在咨询状态。它可以浏览文件但不会做任何更改或运行命令， 直到你批准一个计划。适合探索代码库或需要谨慎处理的情况。

Full Access 模式

Full Access 模式授予 Codex 在你的机器上工作的完全权限，包括网络访问，无需询问。 谨慎使用，只在信任仓库和任务时使用。

切换模式: 使用 /permissions 命令在会话中切换模式。

审批系统

Codex 有三级审批策略来控制它可以做什么：

级别 描述 适用场景
Auto (建议) Codex 可以读取文件、编辑和在工作目录内运行命令。超出范围的操作会询问 日常开发
Read Only 只能浏览文件，不会更改或运行命令 探索、学习代码库
Full Access 完全权限，包括网络访问 仅限受信任的仓库

沙箱模式

Codex 使用沙箱来限制文件系统访问：

  • read-only: 只能读取工作目录（默认）
  • workspace-write: 可以写入工作目录
  • danger-full-access: 无沙箱限制（极其危险）

线程管理

Codex 使用线程来组织工作：

单线程原则

一个线程专注一个任务。如果工作仍是同一个问题，保持在同一线程更好， 因为它保留了推理轨迹。只有当工作真正分支时才分叉。

常用操作

  • /new: 开始新会话
  • /fork: 克隆当前会话（保留原会话，在新线程尝试新方案）
  • /resume: 恢复历史会话
  • /compact: 压缩对话以释放 token

上下文管理

Codex 会话不是简单的聊天历史，而是随着时间积累上下文、决策和行动的工作线程。

上下文层次

  1. 系统指令: Codex 的基础行为准则
  2. AGENTS.md: 项目级配置，位于 ~/.codex/ 或项目根目录
  3. 会话历史: 当前线程的对话记录
  4. @提及文件: 使用 /mention 添加的特定文件上下文

模型选择

Codex 支持多种模型，使用 /model 命令切换：

模型 特点 适用场景
gpt-5.4 最新通用模型 大多数任务
gpt-5.3-codex 专门优化编码 复杂编程任务

推理力度

  • minimal/low: 快速任务
  • medium: 平衡智能和速度（推荐）
  • high/xhigh: 复杂调试和深度推理

多 Agent 工作流

Codex 支持并行多 Agent 来处理复杂任务：

  • 主 Agent 专注于核心问题
  • 子 Agent 处理探索、测试或分类等边界任务
  • 使用 /agent 在 Agent 线程间切换
提示: 多 Agent 功能目前处于实验阶段，使用 /experimental 启用。

Core Concepts

Understanding the core concepts of Codex is key to using it efficiently. This page covers fundamental concepts such as Agent mode, the approval system, and workflows.

What is Codex?

OpenAI Codex is an AI programming assistant built on the GPT-5 series of models, designed specifically for software engineering tasks. Codex is not just a code generator -- it is an Agent capable of:

  • Reading and understanding your codebase
  • Creating and editing files
  • Running commands and tests
  • Performing code reviews
  • Safely executing code in a sandbox environment

Working Modes

Agent Mode (Default)

In Agent mode, Codex can read files, run commands, and write code. It will ask for your approval before performing operations that require it. This is the most commonly used mode and is suitable for most development tasks.

Read Only Mode

Read Only mode keeps Codex in a consultative state. It can browse files but will not make any changes or run commands until you approve a plan. Suitable for exploring codebases or situations requiring caution.

Full Access Mode

Full Access mode grants Codex full permissions to work on your machine, including network access, without asking. Use with caution -- only when you trust the repository and the task.

Switch Modes: Use the /permissions command to switch modes during a session.

Approval System

Codex has a three-tier approval policy to control what it can do:

Level Description Use Case
Auto (Recommended) Codex can read files, edit, and run commands within the working directory. Out-of-scope operations will prompt for approval Daily development
Read Only Can only browse files, will not make changes or run commands Exploring, learning a codebase
Full Access Full permissions, including network access Trusted repositories only

Sandbox Mode

Codex uses sandboxing to restrict file system access:

  • read-only: Can only read the working directory (default)
  • workspace-write: Can write to the working directory
  • danger-full-access: No sandbox restrictions (extremely dangerous)

Thread Management

Codex uses threads to organize work:

Single Thread Principle

One thread focuses on one task. If the work is still about the same issue, it's better to stay in the same thread because it retains the reasoning trace. Only fork when the work truly branches.

Common Operations

  • /new: Start a new session
  • /fork: Clone the current session (keeps the original session, tries a new approach in a new thread)
  • /resume: Resume a previous session
  • /compact: Compact the conversation to free up tokens

Context Management

A Codex session is not just a simple chat history -- it is a working thread that accumulates context, decisions, and actions over time.

Context Hierarchy

  1. System Instructions: The foundational behavior guidelines for Codex
  2. AGENTS.md: Project-level configuration, located in ~/.codex/ or the project root
  3. Session History: The conversation log of the current thread
  4. @Mentioned Files: Specific file context added using /mention

Model Selection

Codex supports multiple models. Use the /model command to switch:

Model Features Use Case
gpt-5.4 Latest general-purpose model Most tasks
gpt-5.3-codex Specifically optimized for coding Complex programming tasks

Reasoning Effort

  • minimal/low: Quick tasks
  • medium: Balance between intelligence and speed (recommended)
  • high/xhigh: Complex debugging and deep reasoning

Multi-Agent Workflows

Codex supports parallel multi-agent workflows for handling complex tasks:

  • The main Agent focuses on the core problem
  • Sub-agents handle peripheral tasks such as exploration, testing, or classification
  • Use /agent to switch between Agent threads
Tip: The multi-agent feature is currently experimental. Enable it with /experimental.

基本概念

Codex を効率よく使うには、基本概念を理解することが重要です。このページでは Agent モード、承認システム、ワークフローなどの土台を整理します。

Codex とは？

OpenAI Codex は GPT-5 系モデルを基盤にした AI プログラミングアシスタントで、ソフトウェア開発タスク向けに設計されています。 Codex は単なるコード生成器ではなく、次のことができる Agent です：

  • コードベースを読み取り理解する
  • ファイルを作成・編集する
  • コマンドやテストを実行する
  • コードレビューを行う
  • サンドボックス環境で安全にコードを実行する

動作モード

Agent Mode（デフォルト）

Agent モードでは、Codex はファイルの読み取り、コマンド実行、コードの書き換えができます。 承認が必要な操作を行う前には確認を求めます。もっとも一般的なモードで、ほとんどの開発タスクに向いています。

Read Only Mode

Read Only モードでは、Codex は相談役として動作します。ファイル閲覧はできますが、計画が承認されるまで変更やコマンド実行は行いません。 コードベースの調査や慎重さが必要な場面に向いています。

Full Access Mode

Full Access モードは、ネットワークアクセスを含め、マシン上での作業権限を Codex に全面的に与えます。 注意して使用してください。信頼できるリポジトリとタスクに限るべきです。

モード切替: セッション中に /permissions コマンドで切り替えられます。

承認システム

Codex には、何を実行できるかを制御する 3 段階の承認ポリシーがあります：

レベル 説明 利用場面
Auto（推奨） 作業ディレクトリ内ではファイルの読み取り・編集・コマンド実行が可能。範囲外の操作は承認を要求する 日常的な開発
Read Only ファイル閲覧のみ可能で、変更やコマンド実行は行わない 調査、コードベース学習
Full Access ネットワークを含む完全権限 信頼できるリポジトリのみ

サンドボックスモード

Codex はサンドボックスでファイルシステムへのアクセス範囲を制限します：

  • read-only: 作業ディレクトリを読むだけ（デフォルト）
  • workspace-write: 作業ディレクトリへの書き込みが可能
  • danger-full-access: サンドボックス制限なし（非常に危険）

スレッド管理

Codex はスレッドを使って作業を整理します：

単一スレッドの原則

1 つのスレッドは 1 つのタスクに集中する のが基本です。同じ課題を進めているなら、同じスレッドに留まるほうが推論の流れを維持できます。本当に作業が分岐するときだけ fork します。

よく使う操作

  • /new: 新しいセッションを開始
  • /fork: 現在のセッションを複製（元を残しつつ別アプローチを新スレッドで試す）
  • /resume: 以前のセッションを再開
  • /compact: 会話を圧縮してトークンを節約

コンテキスト管理

Codex のセッションは単なるチャット履歴ではなく、時間とともに文脈・判断・操作履歴が蓄積される作業スレッドです。

コンテキスト階層

  1. System Instructions: Codex の根本的な挙動ルール
  2. AGENTS.md: ~/.codex/ またはプロジェクトルートに置く、プロジェクト単位の設定
  3. Session History: 現在のスレッドの会話履歴
  4. @ で参照したファイル: /mention などで追加した個別ファイルの文脈

モデル選択

Codex は複数モデルをサポートしています。/model コマンドで切り替えます：

モデル 特徴 向いている用途
gpt-5.4 最新の汎用モデル 大半のタスク
gpt-5.3-codex コーディング向けに最適化 複雑なプログラミング作業

推論強度

  • minimal/low: 短時間で終わるタスク向け
  • medium: 賢さと速度のバランスが良い（推奨）
  • high/xhigh: 複雑なデバッグや深い推論向け

マルチエージェントワークフロー

Codex は複雑なタスクを扱うために、並列マルチエージェントワークフローをサポートしています：

  • メイン Agent は中心課題に集中する
  • サブ Agent は調査、テスト、分類など周辺作業を担当する
  • /agent を使って Agent スレッドを切り替える
ヒント: マルチエージェント機能は現在 experimental 扱いです。/experimental で有効化します。

Conceptos Core

Entender los conceptos core de Codex es clave para usarlo con eficiencia. Esta pagina cubre temas fundamentales como el modo Agent, el sistema de aprobacion y los flujos de trabajo.

Que es Codex?

OpenAI Codex es un asistente de programacion basado en la serie GPT-5, disenado especificamente para tareas de ingenieria de software. Codex no es solo un generador de codigo: es un Agent capaz de:

  • Leer y entender tu base de codigo
  • Crear y editar archivos
  • Ejecutar comandos y pruebas
  • Realizar revisiones de codigo
  • Ejecutar codigo de forma segura dentro de un sandbox

Modos de trabajo

Agent Mode (Predeterminado)

En Agent mode, Codex puede leer archivos, ejecutar comandos y escribir codigo. Pedira tu aprobacion antes de realizar operaciones que la requieran. Es el modo mas utilizado y sirve para la mayoria de las tareas de desarrollo.

Read Only Mode

Read Only mode mantiene a Codex en un estado consultivo. Puede revisar archivos, pero no hara cambios ni ejecutara comandos hasta que apruebes un plan. Es adecuado para explorar repositorios o trabajar con mas cautela.

Full Access Mode

Full Access mode le da a Codex permisos completos en tu maquina, incluido acceso a la red, sin pedir confirmacion. Usalo con cuidado; solo cuando confies en el repositorio y en la tarea.

Cambiar de modo: Usa el comando /permissions para cambiar de modo durante una sesion.

Sistema de aprobacion

Codex tiene una politica de aprobacion de tres niveles para controlar lo que puede hacer:

Nivel Descripcion Caso de uso
Auto (Recomendado) Codex puede leer archivos, editar y ejecutar comandos dentro del directorio de trabajo. Las operaciones fuera del alcance pediran aprobacion Desarrollo diario
Read Only Solo puede examinar archivos; no hace cambios ni ejecuta comandos Exploracion y aprendizaje del repositorio
Full Access Permisos completos, incluido acceso a la red Solo para repositorios confiables

Sandbox mode

Codex usa sandboxing para restringir el acceso al sistema de archivos:

  • read-only: Solo puede leer el directorio de trabajo (predeterminado)
  • workspace-write: Puede escribir dentro del directorio de trabajo
  • danger-full-access: Sin restricciones de sandbox (extremadamente peligroso)

Gestion de hilos

Codex usa hilos para organizar el trabajo:

Principio de un solo hilo

Un hilo se concentra en una sola tarea. Si el trabajo sigue siendo sobre el mismo problema, conviene permanecer en el mismo hilo porque conserva el rastro de razonamiento. Solo deberias hacer fork cuando el trabajo se bifurca de verdad.

Operaciones comunes

  • /new: Iniciar una nueva sesion
  • /fork: Clonar la sesion actual (mantiene la sesion original y prueba otro enfoque en un hilo nuevo)
  • /resume: Reanudar una sesion anterior
  • /compact: Compactar la conversacion para liberar tokens

Gestion del contexto

Una sesion de Codex no es solo un historial de chat. Es un hilo de trabajo que acumula contexto, decisiones y acciones con el tiempo.

Jerarquia del contexto

  1. System Instructions: Las reglas base de comportamiento para Codex
  2. AGENTS.md: Configuracion a nivel de proyecto ubicada en ~/.codex/ o en la raiz del proyecto
  3. Session History: El historial de conversacion del hilo actual
  4. Archivos mencionados con @: Contexto especifico agregado mediante /mention

Seleccion de modelo

Codex admite varios modelos. Usa el comando /model para cambiar:

Modelo Caracteristicas Caso de uso
gpt-5.4 Modelo general mas reciente La mayoria de las tareas
gpt-5.3-codex Optimizado especificamente para programacion Tareas de programacion complejas

Esfuerzo de razonamiento

  • minimal/low: Tareas rapidas
  • medium: Equilibrio entre inteligencia y velocidad (recomendado)
  • high/xhigh: Depuracion compleja y razonamiento profundo

Flujos multi-agent

Codex admite flujos paralelos con varios agentes para manejar tareas complejas:

  • El Agent principal se centra en el problema core
  • Los subagentes se ocupan de tareas perifericas como exploracion, pruebas o clasificacion
  • Usa /agent para cambiar entre hilos de Agent
Consejo: La funcion multi-agent sigue siendo experimental. Activalo con /experimental.
