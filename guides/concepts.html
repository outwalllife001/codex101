核心概念

理解 Codex 的核心概念是高效使用它的关键。本页面介绍 Agent 模式、审批系统、工作流等基础概念。

什么是 Codex？

OpenAI Codex 是一个 AI 编程助手，它基于 GPT-5 系列模型，专门用于软件工程任务。 Codex 不仅仅是一个代码生成器，它是一个Agent（智能体），能够：

  • 阅读和理解你的代码库
  • 创建和编辑文件
  • 运行命令和测试
  • 进行代码审查
  • 在沙箱环境中安全执行代码

工作模式

Agent 模式（默认）

在 Agent 模式下，Codex 可以读取文件、运行命令、编写代码。它会在执行需要批准的操作前询问你。 这是最常用的模式，适合大多数开发任务。

Read Only 模式

Read Only 模式保持 Codex 在咨询状态。它可以浏览文件但不会做任何更改或运行命令， 直到你批准一个计划。适合探索代码库或需要谨慎处理的情况。

Full Access 模式

Full Access 模式授予 Codex 在你的机器上工作的完全权限，包括网络访问，无需询问。 谨慎使用，只在信任仓库和任务时使用。

切换模式: 使用 /permissions 命令在会话中切换模式。

审批系统

Codex 有三级审批策略来控制它可以做什么：

级别 描述 适用场景
Auto (建议) Codex 可以读取文件、编辑和在工作目录内运行命令。超出范围的操作会询问 日常开发
Read Only 只能浏览文件，不会更改或运行命令 探索、学习代码库
Full Access 完全权限，包括网络访问 仅限受信任的仓库

沙箱模式

Codex 使用沙箱来限制文件系统访问：

  • read-only: 只能读取工作目录（默认）
  • workspace-write: 可以写入工作目录
  • danger-full-access: 无沙箱限制（极其危险）

线程管理

Codex 使用线程来组织工作：

单线程原则

一个线程专注一个任务。如果工作仍是同一个问题，保持在同一线程更好， 因为它保留了推理轨迹。只有当工作真正分支时才分叉。

常用操作

  • /new: 开始新会话
  • /fork: 克隆当前会话（保留原会话，在新线程尝试新方案）
  • /resume: 恢复历史会话
  • /compact: 压缩对话以释放 token

上下文管理

Codex 会话不是简单的聊天历史，而是随着时间积累上下文、决策和行动的工作线程。

上下文层次

  1. 系统指令: Codex 的基础行为准则
  2. AGENTS.md: 项目级配置，位于 ~/.codex/ 或项目根目录
  3. 会话历史: 当前线程的对话记录
  4. @提及文件: 使用 /mention 添加的特定文件上下文

模型选择

Codex 支持多种模型，使用 /model 命令切换：

模型 特点 适用场景
gpt-5.4 最新通用模型 大多数任务
gpt-5.3-codex 专门优化编码 复杂编程任务

推理力度

  • minimal/low: 快速任务
  • medium: 平衡智能和速度（推荐）
  • high/xhigh: 复杂调试和深度推理

多 Agent 工作流

Codex 支持并行多 Agent 来处理复杂任务：

  • 主 Agent 专注于核心问题
  • 子 Agent 处理探索、测试或分类等边界任务
  • 使用 /agent 在 Agent 线程间切换
提示: 多 Agent 功能目前处于实验阶段，使用 /experimental 启用。
