多 Agent 协作开发

高级 2 小时

场景

开发一个完整的 SaaS 应用，包含前端、后端、文档和测试。使用多个 Agent 并行处理不同模块。

启用实验性功能

/experimental

任务分解

主 Agent - 核心功能

> 创建项目基础架构，包含：
> 1. 项目目录结构
> 2. 共享类型定义
> 3. 开发环境配置
> 4. CI/CD 配置文件

子 Agent 1 - 后端 API

# 在新线程中
> 开发用户认证 API（注册、登录、JWT）
> 使用 Node.js + Express + TypeScript
> 包含完整的单元测试

子 Agent 2 - 前端界面

# 在新线程中
> 创建 React 前端应用
> 实现登录/注册页面
> 使用 Tailwind CSS 美化界面

子 Agent 3 - 文档

# 在新线程中
> 编写 API 文档（OpenAPI/Swagger）
> 创建 README 和部署指南

切换和管理 Agent

# 查看所有线程
/agent

# 切换到后端线程
/agent 1

# 检查进度
> 进度如何？有什么阻塞吗？

# 切换到前端线程
/agent 2

# 合并前审查
/review

协作模式

Agent 职责 输出
主 Agent 架构、协调 项目骨架、类型定义
后端 Agent API 开发 Routes, Controllers, Services
前端 Agent UI 开发 Components, Pages, Hooks
文档 Agent 文档编写 README, API docs, Guides

最佳实践

  • ✅ 主 Agent 先定义接口和数据结构
  • ✅ 子 Agent 并行工作时避免冲突文件
  • ✅ 定期同步进度
  • ✅ 最后统一进行代码审查
