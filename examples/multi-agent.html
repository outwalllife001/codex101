多 Agent 协作开发

高级 2 小时

场景

开发一个完整的 SaaS 应用，包含前端、后端、文档和测试。使用多个 Agent 并行处理不同模块。

启用实验性功能

/experimental

任务分解

主 Agent - 核心功能

> 创建项目基础架构，包含：
> 1. 项目目录结构
> 2. 共享类型定义
> 3. 开发环境配置
> 4. CI/CD 配置文件

子 Agent 1 - 后端 API

# 在新线程中
> 开发用户认证 API（注册、登录、JWT）
> 使用 Node.js + Express + TypeScript
> 包含完整的单元测试

子 Agent 2 - 前端界面

# 在新线程中
> 创建 React 前端应用
> 实现登录/注册页面
> 使用 Tailwind CSS 美化界面

子 Agent 3 - 文档

# 在新线程中
> 编写 API 文档（OpenAPI/Swagger）
> 创建 README 和部署指南

切换和管理 Agent

# 查看所有线程
/agent

# 切换到后端线程
/agent 1

# 检查进度
> 进度如何？有什么阻塞吗？

# 切换到前端线程
/agent 2

# 合并前审查
/review

协作模式

Agent 职责 输出
主 Agent 架构、协调 项目骨架、类型定义
后端 Agent API 开发 Routes, Controllers, Services
前端 Agent UI 开发 Components, Pages, Hooks
文档 Agent 文档编写 README, API docs, Guides

最佳实践

  • 主 Agent 先定义接口和数据结构
  • 子 Agent 并行工作时避免冲突文件
  • 定期同步进度
  • 最后统一进行代码审查

Multi-Agent Collaborative Development

Advanced 2 hours

Scenario

Build a complete SaaS application including frontend, backend, documentation, and tests. Use multiple Agents to handle different modules in parallel.

Enable Experimental Features

/experimental

Task Breakdown

Main Agent - Core Functionality

> Create the project scaffolding, including:
> 1. Project directory structure
> 2. Shared type definitions
> 3. Development environment configuration
> 4. CI/CD configuration files

Sub Agent 1 - Backend API

# In a new thread
> Develop user authentication API (register, login, JWT)
> Use Node.js + Express + TypeScript
> Include complete unit tests

Sub Agent 2 - Frontend UI

# In a new thread
> Create a React frontend application
> Implement login/registration pages
> Style the UI with Tailwind CSS

Sub Agent 3 - Documentation

# In a new thread
> Write API documentation (OpenAPI/Swagger)
> Create README and deployment guide

Switching and Managing Agents

# View all threads
/agent

# Switch to backend thread
/agent 1

# Check progress
> How's the progress? Any blockers?

# Switch to frontend thread
/agent 2

# Pre-merge review
/review

Collaboration Model

Agent Responsibility Output
Main Agent Architecture, Coordination Project scaffold, Type definitions
Backend Agent API Development Routes, Controllers, Services
Frontend Agent UI Development Components, Pages, Hooks
Docs Agent Documentation README, API docs, Guides

Best Practices

  • Have the main Agent define interfaces and data structures first
  • Avoid file conflicts when sub Agents work in parallel
  • Sync progress regularly
  • Conduct a unified code review at the end

マルチ Agent 協調開発

上級 2時間

シナリオ

フロントエンド、バックエンド、ドキュメント、テストを含む完全な SaaS アプリを開発します。複数の Agent を使って別々のモジュールを並列に進めます。

実験的機能を有効にする

/experimental

タスクの分解

メイン Agent - コア機能

> 次を含むプロジェクトの土台を作成してください。
> 1. プロジェクトのディレクトリ構成
> 2. 共通の型定義
> 3. 開発環境の設定
> 4. CI/CD 設定ファイル

サブ Agent 1 - バックエンド API

# 新しいスレッドで
> ユーザー認証 API（登録、ログイン、JWT）を開発してください
> Node.js + Express + TypeScript を使用してください
> 完全な単体テストを含めてください

サブ Agent 2 - フロントエンド UI

# 新しいスレッドで
> React フロントエンドアプリを作成してください
> ログイン/登録ページを実装してください
> Tailwind CSS で UI を整えてください

サブ Agent 3 - ドキュメント

# 新しいスレッドで
> API ドキュメント（OpenAPI/Swagger）を書いてください
> README とデプロイ手順書を作成してください

Agent の切り替えと管理

# すべてのスレッドを表示
/agent

# バックエンドスレッドへ切り替え
/agent 1

# 進捗確認
> 進捗はどうですか？何か詰まっていますか？

# フロントエンドスレッドへ切り替え
/agent 2

# マージ前レビュー
/review

協調モデル

Agent 役割 成果物
メイン Agent 設計、調整 プロジェクト骨格、型定義
バックエンド Agent API 開発 Routes, Controllers, Services
フロントエンド Agent UI 開発 Components, Pages, Hooks
ドキュメント Agent 文書作成 README, API docs, Guides

ベストプラクティス

  • まずメイン Agent がインターフェースとデータ構造を定義する
  • サブ Agent を並列で動かす際はファイル競合を避ける
  • 定期的に進捗を同期する
  • 最後に統一したコードレビューを行う

Desarrollo colaborativo con multiples Agent

Avanzado 2 horas

Escenario

Construye una aplicacion SaaS completa con frontend, backend, documentacion y pruebas. Usa varios Agent para trabajar en paralelo sobre modulos distintos.

Activar funciones experimentales

/experimental

Desglose de tareas

Agent principal - Funcionalidad central

> Crea la base del proyecto, incluyendo:
> 1. Estructura de directorios del proyecto
> 2. Definiciones de tipos compartidos
> 3. Configuracion del entorno de desarrollo
> 4. Archivos de configuracion de CI/CD

Sub Agent 1 - Backend API

# En un hilo nuevo
> Desarrolla la API de autenticacion de usuarios (registro, login, JWT)
> Usa Node.js + Express + TypeScript
> Incluye pruebas unitarias completas

Sub Agent 2 - Interfaz frontend

# En un hilo nuevo
> Crea una aplicacion frontend con React
> Implementa las paginas de login y registro
> Mejora la interfaz con Tailwind CSS

Sub Agent 3 - Documentacion

# En un hilo nuevo
> Escribe la documentacion de la API (OpenAPI/Swagger)
> Crea el README y la guia de despliegue

Cambiar y gestionar Agent

# Ver todos los hilos
/agent

# Cambiar al hilo del backend
/agent 1

# Revisar progreso
> Como va el progreso? Hay algun bloqueo?

# Cambiar al hilo del frontend
/agent 2

# Revision antes de fusionar
/review

Modelo de colaboracion

Agent Responsabilidad Salida
Agent principal Arquitectura y coordinacion Estructura base del proyecto, definiciones de tipos
Agent de backend Desarrollo de API Routes, Controllers, Services
Agent de frontend Desarrollo de UI Components, Pages, Hooks
Agent de documentacion Redaccion de documentacion README, API docs, Guides

Buenas practicas

  • Haz que el Agent principal defina primero interfaces y estructuras de datos
  • Evita conflictos de archivos cuando los sub Agent trabajan en paralelo
  • Sincroniza el progreso de forma regular
  • Realiza una revision de codigo unificada al final
