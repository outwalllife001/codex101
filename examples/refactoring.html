代码重构与优化

高级 1 小时

场景

你接手了一个遗留项目，需要将其从 JavaScript 迁移到 TypeScript，优化性能，并添加测试覆盖。

步骤 1：创建 Checkpoint

git add -A && git commit -m "checkpoint: before refactoring"

步骤 2：分析和规划

/plan

分析 @src/ 目录下的所有 JavaScript 文件，创建一个详细的 TypeScript 迁移计划，包括：
1. 文件迁移顺序（按依赖关系）
2. 需要添加的类型定义
3. 潜在的类型安全问题
4. 需要重构的代码模式

步骤 3：渐进式迁移

迁移工具函数

> 将 @src/utils/ 下的工具函数转换为 TypeScript
> 添加完整的类型定义
> 确保所有函数都有返回类型注解

迁移数据层

> 将 @src/models/ 和 @src/services/ 转换为 TypeScript
> 定义所有数据接口
> 使用严格类型检查

迁移 UI 组件

> 将 @src/components/ 转换为 TypeScript React 组件
> 为 Props 定义接口
> 修复任何类型错误

步骤 4：性能优化

> 分析 @src/components/List.tsx 的性能问题
> 优化渲染，添加 useMemo 和 useCallback
> 实现虚拟滚动处理大数据列表

步骤 5：添加测试

> 为核心业务逻辑编写单元测试
> 添加 React 组件测试（React Testing Library）
> 确保测试覆盖率达到 80% 以上

步骤 6：代码审查

/review

关键技巧

  • ✅ 使用 /plan 制定详细计划
  • ✅ 分批迁移，一次一个模块
  • ✅ 频繁使用 /diff 查看变更
  • ✅ 完成后进行全面 /review
